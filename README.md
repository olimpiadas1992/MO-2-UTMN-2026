# Репозиторий для практик по машинному обучению

## Описание проекта

Данный репозиторий содержит практические работы по дисциплине «Машинное обучение» (Machine Learning). В качестве исходного набора данных используется датасет [`aug_train.csv`](aug_train.csv), содержащий информацию о сотрудниках и их карьерных перемещениях.

## Структура проекта

```
├── aug_train.csv                    # Исходный набор данных
├── features.ipynb                   # Очистка данных и создание признаков
├── 00-Cross-Validation.ipynb        # Практика: кросс-валидация
├── artifacts/                       # Папка с подготовленными артефактами
│   ├── X_prepared_sample.csv        # Подготовленные данные для обучения
│   ├── missing_summary.csv          # Сводка по пропущенным значениям
│   ├── outliers_summary.csv         # Сводка по выбросам
│   ├── report_autofill.json         # Отчёт об автозаполнении
│   └── structure_summary.csv        # Сводка по структуре данных
├── requirements.txt                 # Зависимости проекта
└── README.md                        # Документация проекта
```

## Описание файлов

### Исходные данные

- **[`aug_train.csv`](aug_train.csv)** — основной набор данных, содержащий информацию о сотрудниках, их характеристиках и целевой переменной (статус поиска работы).

### Обработка данных

- **[`features.ipynb`](features.ipynb)** — ноутбук, содержащий код для очистки данных и создания признаков. Включает:
  - Анализ пропущенных значений
  - Обработку выбросов
  - Автозаполнение пропусков
  - Создание новых признаков
  - Формирование подготовленного набора данных

### Артефакты

Папка [`artifacts/`](artifacts/) содержит результаты обработки данных:

- **[`X_prepared_sample.csv`](artifacts/X_prepared_sample.csv)** — подготовленный набор данных, готовый для обучения моделей машинного обучения и использования в последующих практиках.
- **[`missing_summary.csv`](artifacts/missing_summary.csv)** — сводная информация о пропущенных значениях в исходном датасете.
- **[`outliers_summary.csv`](artifacts/outliers_summary.csv)** — информация о выбросах, обнаруженных в данных.
- **[`report_autofill.json`](artifacts/report_autofill.json)** — отчёт о процедуре автозаполнения пропущенных значений.
- **[`structure_summary.csv`](artifacts/structure_summary.csv)** — описание структуры исходного набора данных.

### Описание признаков в X_prepared_sample.csv

Подготовленный набор данных содержит следующие признаки:

#### Числовые признаки:
- **city_development_index** — индекс развития города
- **experience** — опыт работы (в годах)
- **last_new_job** — время с последней смены работы (в годах)
- **training_hours** — количество часов обучения
- **target** — целевая переменная (статус поиска работы: 1 — ищет работу, 0 — не ищет)

#### Бинарные признаки:
- **no_job_info** — отсутствие информации о текущей работе (1 — да, 0 — нет)
- **is_university_enrolled** — зачисление в университет (1 — да, 0 — нет)
- **relevent_experience_No relevent experience** — отсутствие релевантного опыта (True/False)

#### Категориальные признаки (закодированы методом one-hot encoding):
- **city_city_\*** — город проживания (более 100 уникальных городов)
- **company_type_\*** — тип компании (Funded Startup, NGO, Other, Public Sector, Pvt Ltd, Unknown)
- **company_size_\*** — размер компании (<10, 50-99, 100-500, 500-999, 1000-4999, 5000-9999, 10000+, Unknown)
- **enrolled_university_\*** — статус зачисления в университет (Part time course, no_enrollment, not_stated)
- **education_level_\*** — уровень образования (1-5)
- **major_discipline_\*** — основная дисциплина (Business Degree, Humanities, No Major, Other, STEM, not_stated)
- **gender_\*** — пол (Male, Other, not_stated)

### Практические работы

- **[`00-Cross-Validation.ipynb`](00-Cross-Validation.ipynb)** — практическая работа по кросс-валидации моделей машинного обучения.

## Установка и запуск

### Требования

Все необходимые библиотеки перечислены в файле [`requirements.txt`](requirements.txt).

### Установка зависимостей

```bash
pip install -r requirements.txt
```

### Запуск ноутбуков

1. Убедитесь, что установлены все зависимости из [`requirements.txt`](requirements.txt)
2. Откройте нужный Jupyter-ноутбук (`.ipynb`)
3. Запустите ячейки последовательно
